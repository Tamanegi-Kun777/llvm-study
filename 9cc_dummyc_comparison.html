9cc と DummyC(きつねさん) 対照表

文法要素の対照表

DummyC(きつねさん) 9cc 意味
TranslationUnitprogramプログラム全体(ファイル全体)
ExternalDeclarationis_function()による判定と、function(), global_var()の呼び出し関数宣言または、関数定義
FunctionDeclarationfunction()の中のif (consume(";"))のケース宣言のみの関数(例: int printf();)
FunctionDefinitionfunction()の中のfn->body = stmt();のケース本体を持つ、関数の定義
Prototypefunction()の中の、型・関数名・引数リストを読む部分関数の型・名前・引数の並び
FunctionStatementstmt()の中の{の処理(ND_BLOCK)関数の中身({ ... })
VariableDeclarationstmt()の中のpeek_type_keyword()の分岐int a;のような、変数宣言
Statementstmt()1つの文
ExpressionStatementstmt()の中の、通常の式文a = 1;のような、式+;
JumpStatementstmt()の中のTK_RETURNの分岐return文
AssignmentExpressionassign()代入式(a = 1のような)
AdditiveExpressionadd()足し算・引き算
MultiplicativeExpressionmul()掛け算・割り算
PostfixExpressionprimary()の中の、関数呼び出しの部分変数や数値の後に(が続く、関数呼び出し
PrimaryExpressionprimary()の中の、数値・変数の部分数値・変数など、それ以上分解できない、最小単位の式
9ccの文法の階層(expr = mul (...)* / mul = primary (...)*)と、DummyCのAdditiveExpression → MultiplicativeExpression → PostfixExpression → PrimaryExpressionという階層は、全く同じ優先順位の考え方に基づいています。

補足:名前の付け方の傾向

DummyC(きつねさん)は Expression(式)、Statement(文)、Declaration(宣言)という英語の文法用語を、そのままクラス名・関数名に使っています。 これは多くのプログラミング言語の教科書や実際のコンパイラの実装でよく使われる標準的な命名の慣習です。 9ccはより短くシンプルな名前(expr, add, mul, primary)を使っていましたが、指し示している概念自体は共通しています。
